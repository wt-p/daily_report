## わかったこと
### Remix
- Remixでは frontend/app/routes 配下のファイルがURLに対応する画面コンポーネントとなる
- $ が付いたファイルは動的ルーティングを表し、/checks/:id のようなパラメータ付きURLに対応する

### async / await
- async は関数が非同期処理を含むことを示す宣言
- await は非同期処理の完了を待つためのキーワード
- JavaScriptは基本的に処理を待たず次の処理へ進むため、async と await を組み合わせることで処理の完了を待つことができる


### TypeScript
- type Props はReactコンポーネントが受け取るpropsの型定義
- ? が付いているプロパティは optional（省略可能）であることを意味する


### API処理の追い方
- ブラウザの開発者ツールの Networkタブ でAPIリクエストを特定する
- 今回の環境ではOpenAPIを使用しているため、spec.yml → api_*.go（Routes / Controller） の順に確認することで、フロントエンドからバックエンドの処理の流れを追うことができる
