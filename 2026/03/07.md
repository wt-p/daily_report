## わかったこと
- 昨日実施していたが曖昧だったspec.yml 以降の処理の流れについて整理<br>
→ spec.yml の operationId を起点に
api_*.go → OrderedRoutes() → c.XxxYyy → XxxYyy（Controller） → usecaseの順でバックエンド処理を追えることを確認した

- OrderedRoutes() で API エンドポイントと Controller メソッドのルーティングが登録される
- Goの基本的な構造について以下の点を理解
  - c.XxxYyy のような呼び出しは func (c *XxxAPIController) XxxYyy に紐付く
  - func (c *Controller) のように書くことで、Javaでクラスにメソッドを定義するのと同じような形で実装できる
  - type Xxx struct {} がJavaのクラス定義に相当
  - Java のようにimplements を明示しない
  - コンストラクタがないため NewXxx のような関数で初期化する
