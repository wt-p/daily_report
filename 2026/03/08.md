## わかったこと
- Reactではデータ（props）は基本的に 親コンポーネント → 子コンポーネントへ一方向に流れる
- コンポーネントに渡した値は propsオブジェクトとして受け取られる
- {...props} を使うことで propsオブジェクトの中身を展開して子コンポーネントへ渡せる
- ラッパーコンポーネントでは 受け取ったpropsを子へ渡しつつ、一部のpropsだけ加工して渡すことができる
- Tailwindでは ! を付けることで !important と同じ意味になりCSSの優先度が高くなる
- JSXでは {} を使うことで JavaScriptの値や式を書くことができる
- JSXの中では if 文は使えないため、条件分岐は三項演算子で書くことが多い
- 三項演算子をネストすることで if / else if / else と同じ条件分岐を表現できる
 - JSX内の条件分岐が複雑になる場合は、表示する要素を変数に切り出すと可読性が上がる
- Reactでは null を返すとコンポーネントは描画されない
- value != null は null と undefined の両方をまとめて判定するための書き方で、JS / React でよく使われる
- value != null とすることで 0 や false などの有効な値を誤って除外せずにチェックできる

- Nullish Coalescing
  - value ?? 1 は valueがnull / undefinedのときだけ1を使う
  - 0 や ‘’ は有効な値として扱われる

- Hook
  - useXxx のように use で始まる関数は Hook（フック）
  - Reactが提供するものは 組み込みHook（useState, useEffect など）、自分で作るものは カスタムHook
  - コンポーネントからロジックを切り出し、再利用できるようにする仕組み
